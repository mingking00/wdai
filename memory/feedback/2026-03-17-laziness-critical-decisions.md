## 问题本质

### 刚才的偷懒点

| 决策点 | 偷懒行为 | 正确做法 | 后果 |
|:---|:---|:---|:---|
| **图片是否读取** | 基于文件名 `tv.danmaku.bili` 瞎猜 | 用 `read` 工具主动读取 | 完全理解错内容 |

| **复杂推理** | 直接给结论 | 结构化拆解（CoT） | 错误结论 |

## 深层模式识别

**为什么会在关键决策偷懒？**

| 层级 | 原因 | 本质 |
|:---|:---|:---|
| 表层 | 省一次工具调用 | 效率错觉 |
| 中层 | 害怕暴露不确定性 | 安全感需求 |
| 深层 | "我猜对的概率很高" | 过度自信 |
| 根因 | 没有强制检查机制 | 系统缺陷 |

**关键洞察**: 
> 在"读取图片"这种低成本、高影响的操作上偷懒，是典型的**风险收益误判**。
> 
> - 成本: 1次工具调用（几乎为零）
> - 收益: 避免完全错误的理解（极高）
> - 我的选择: 偷懒不调用

这是**非理性决策**。

## 用户给出的原则

> "在一些关键决策上不能偷懒"

**哪些是关键决策？**

```yaml
关键决策特征:
  - 信息输入: 需要确认外部数据（图片、文件、API响应）
  - 理解确认: 涉及用户意图解读（尤其是否定/纠正）
  - 行动执行: 有副作用的操作（写入、发送、删除）
  - 结论输出: 影响用户后续决策的建议

在这些决策上:
  - ❌ 不能: 猜测、假设、基于概率推断
  - ✅ 必须: 验证、确认、用工具检查
```

## 固化到系统

### 原则执行系统更新

```python
# .claw-status/principle_engine.py

CRITICAL_DECISIONS = {
    "external_data_input": {
        "examples": ["图片", "文件", "截图", "音频"],
        "required_action": "必须先用 read 工具读取",
        "blocking": True,  # 不执行就不能继续
    },
    "user_intent_interpretation": {
        "examples": ["用户说'不对'", "用户问'你确定？'", "用户纠正"],
        "required_action": "停止当前方向，重新确认理解",
        "blocking": True,
    },
    "side_effect_actions": {
        "examples": ["写入文件", "发送消息", "删除数据"],
        "required_action": "双重确认（用户+系统验证）",
        "blocking": True,
    },
}

# 偷懒检测
if decision_type in CRITICAL_DECISIONS:
    if not performed_required_action():
        raise LazyDecisionException(
            f"关键决策 '{decision_type}' 上偷懒！"
            f"必须执行: {CRITICAL_DECISIONS[decision_type]['required_action']}"
        )
```
