### 关键发现

**Coding Plan key (`sk-cp-`) 必须使用Anthropic协议：**

| 协议 | Base URL | 适用Key |
|------|----------|---------|
| **OpenAI兼容** | `api.minimaxi.com/v1` | `sk-api-...` (普通key) |
| **Anthropic兼容** | `api.minimaxi.com/anthropic` | `sk-cp-...` (Coding Plan) ❌ |

**错误排查**：
- ❌ `api.minimaxi.com/v1` + `sk-cp-` = 10021 invalid api key
- ❌ `api.minimaxi.chat/v1` + `sk-cp-` = 2049 invalid api key
- ✅ `api.minimaxi.com/anthropic` + `sk-cp-` = 成功！

## EMPO² + MemRL 记忆系统升级 ✅

### 核心改进

将EMPO²论文的四个核心机制整合到MemRL记忆系统：

| EMPO²机制 | 实现方式 | 状态 |
|-----------|---------|------|
| **Self-Generated Tips** | 自动从失败提取指导提示 | ✅ |
| **Hybrid Rollout** | 自适应选择有/无记忆模式 | ✅ |
| **On/Off-Policy Update** | 直接反馈+知识蒸馏 | ✅ |
| **Knowledge Distillation** | 高频Tips自动内化 | ✅ |

### 架构对比

**原有MemRL**:
```
记忆 = (z, e, Q)
```

**EMPO²增强版**:
```
记忆 = (z, e, Q, Tips, Rules)
+ Tips: 自我生成的指导提示
+ Rules: 已内化的系统规则
```

### 自动功能

**1. 错误→Tips**:
```
API限流错误 
→ 自动生成Tip: "⏳ 限流错误 - 添加重试逻辑或降低请求频率"
```

**2. 高频Tips→内化**:
```
Tip使用5次 + 有效性>0.7
→ 自动添加到auto_internalized_rules.md
→ 后续无需外部记忆即可使用
```

**3. 探索奖励**:
```
全新任务类型 → 探索奖励 x2
困难任务(成功率<30%) → 探索奖励 x1
```
