## 🔍 发现的模式

### 成功模式

#### 1. 深度内化循环
**模式**: 学习 → 反思 → 工具化 → 验证

**案例**: Claude Code视频学习
- 观看视频 → 提取核心启示 → 创建工作流工具 → 在后续任务中验证
- 结果: 3个核心工具 (optimized_workflow.py, conversation_workflow.py, checklist.py)

**可复用**: 任何学习场景都应强制走完这个循环

#### 2. 渐进式架构演进
**模式**: 简单规则 → 遇到问题 → 被迫改进

**案例**: Vibero阅读器开发
- Phase 1: Flask后端 → Phase 2: 整书翻译 → Phase 3: 无服务器方案
- 每个阶段都是被用户需求"逼"出来的

**启发**: 不要预先设计完美架构，从MVP开始让真实需求驱动演进

#### 3. 系统性调研框架
**模式**: 多维度搜索 → 结构化分类 → 对比分析 → 提取洞察

**案例**: GitHub项目调研
- 使用多个关键词组合搜索
- 按技术栈/用途分类
- 排除超额项目，聚焦目标范围
- 输出Top 5推荐清单

#### 4. 技术可行性诚实评估
**模式**: 热情构思 → 用户挑战 → 诚实修正 → 务实定位

**案例**: 智能家居项目
- 初始方案过于理想化 → 被指问题 → 诚实修正 → 聚焦可实现的核心创新

**关键学习**: 技术项目必须直面可行性约束，诚实比乐观更有价值

### 失败/改进模式

#### 1. 沟通效率问题
**问题**: 截图传达不清、登录流程复杂、假设用户理解

**改进**: 
- 确认接收 + 文字描述备份
- 提前列出多种方案让用户选择
- 每一步都提供替代方案

**工具化**: 检查清单中加入"沟通确认"项

#### 2. 过度推断风险
**问题**: 从单一案例推断通用规律

**案例**: 看到Claude Code简单架构成功 → 想推翻所有复杂设计

**修正**: 
- 添加"验证先于推广"铁律到SOUL.md
- 强制执行：小规模测试 → 对照实验 → 文献查证 → 统计显著 → 用户反馈

### 技巧2: 渐进式MVP开发流程

**触发条件**: 任何需要从0构建的项目

**执行步骤**:
1. Phase 1: 核心功能，能跑通一条主流程
2. Phase 2: 用户使用后，收集反馈
3. Phase 3: 根据真实痛点，选择性增强
4. 每个Phase都有明确的成功标准
5. 不达标准不进入下一阶段

**关键原则**: 让真实需求"逼"出架构，而不是预先设计

**验证状态**: ✅ Vibero阅读器3阶段演进验证

**触发条件**: 技术方案设计、项目规划

**检查清单**:
