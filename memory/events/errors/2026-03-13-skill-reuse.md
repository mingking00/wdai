## 2026-03-13: 已有能力优先检查缺失

### 事件
开发ClawFlow Research Pipeline时：
- ❌ **错误**: 想实现搜索 → 在`SkillNode`里重写`_kimi_search()`
- ✅ **正确**: 直接用工作流调用`"skill": "kimi_search"`节点

### 根本原因
**思维盲区**: 陷入"写代码实现"模式，而非"组合现有工具"模式

### 影响
- 浪费时间重写已有功能
- 冗余代码增加维护负担
- 错过更简单优雅的方案

### 纠正措施
**已内化到核心原则**:
1. 执行任何任务前，自动扫描可用资源
2. 查记忆 → 查文档 → 查技能 → 问"能用现有能力解决吗？"
3. 默认假设：已有能力 > 新写代码

### 验证方式
下次遇到类似场景时，检查：
- [ ] memory_search("相关技能和工具")
- [ ] read TOOLS.md
- [ ] ls ~/.openclaw/skills/
- [ ] 问"能用现有技能组合解决吗？"

### 教训
**不要造轮子，要组合轮子。**
