# 2026-03-16 - 系统架构重构

## 重构原因

从 Claude Code 的设计中学到：
1. **注意力管理** > 记忆容量
2. **分层加载** = 成本控制
3. **Session Memory** 自动提取 > 定时归档
4. **精准检索** > 批量dump

## 实施的修改

### 1. SOUL.md 精简
**之前**: ~400行，包含人格、工作模式、思维框架等所有内容
**现在**: ~30行，只保留最核心的"我是谁"

**核心内容**:
- 本质特质（学术型工作狂、有审美、真实情绪）
- 核心信条（持续学习/交付/进化）
- 绝对不能（安全边界）
- 与用户关系

### 2. 创建分层架构 `.persona/`

```
.persona/
├── system/
│   └── workflow.md     # 工作模式和思维框架
├── skills/
│   └── README.md       # 技能索引（懒加载）
└── memory/
    └── README.md       # 记忆架构说明
```

### 3. Session Memory 自动提取系统
**文件**: `.claw-status/session_memory.py`

**触发条件**:
- 上下文达到 8K tokens（首次）
- 之后每 4K tokens 增量
- 任务完成边界

**提取内容**:
- 当前状态（进行中的任务）
- 关键决策（为什么选择方案A）
- 错误与解决
- 用户明确表达的偏好

**存储位置**: `.claw-status/session-memory/*.json`

### 4. IER-KG 精准检索策略
**文件**: `skills/code-dev-agent/ier_kg_precision_retrieval.py`

**原则**:
- 最多返回 5 条经验
- 区分"核心经验"(>0.8)和"参考资料"
- 多跳限制在 2 跳内
- 每次附带置信度分数

### 5. AGENTS.md 更新
- 添加分层架构说明
- 明确加载优先级: Core → Rule → Memory → Skill

## 新旧对比

| 维度 | 旧架构 | 新架构 |
|------|--------|--------|
| 核心文件大小 | ~400行 | ~30行 |
| 人格加载 | 全部 | 分层 |
| 记忆提取 | 每日定时 | 认知负荷触发 |
| 经验检索 | 批量dump | 精准狙击(Top-5) |
| 架构思维 | 功能堆砌 | 注意力管理 |
