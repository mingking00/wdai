## 视频到知识图谱自动化工具开发

### 项目概述
开发了从B站视频到RAG-Anything知识图谱的自动化流程脚本。

### 核心思路
- 用字幕语义分析代替均匀采样，识别真正重要的时刻
- 字幕说重点时提取关键帧，精准打击而非盲目采样
- 输出KG JSON格式，可直接导入RAG-Anything

### 创建的脚本

| 脚本 | 位置 | 功能 |
|------|------|------|
| video_to_kg.py | `.tools/video_to_kg.py` | 主脚本，完整自动化流程 |
| test_video_to_kg.py | `.tools/test_video_to_kg.py` | 测试脚本，验证逻辑 |
| demo_kimi_k2_video.py | `.tools/demo_kimi_k2_video.py` | 演示脚本，模拟真实视频 |

### 处理流程
```
视频 → 字幕提取(yt-dlp/whisper) → 语义分析 → 关键帧提取(ffmpeg) → KG JSON
```

### 关键帧识别策略
- 概念首次出现：新术语、新概念的首次解释
- 重要结论：总结性陈述、关键发现
- 对比分析：不同方案的对比、优缺点
- 关键词评分：核心、总结、对比、创新等加权

### 测试结果
使用 `[LLM Architect] 08 Dive into K2.5` 视频模拟数据测试：
- 10条字幕 → 识别7个关键时间点
- 最高得分：总结段落(21分)、架构设计(13分)
- 输出：KG JSON + Markdown摘要 + AI分析提示词

### 限制与解决
- **限制**：B站反爬限制，yt-dlp直接下载失败
- **解决**：可使用本地视频文件、添加Cookie、或手动下载后处理

### 输出文件位置
```
demo_output/
├── BV1nvcuz5Ewj_kg.json          # 知识图谱导入文件
├── BV1nvcuz5Ewj_summary.md       # 可视化摘要
└── BV1nvcuz5Ewj_analysis_prompt.txt  # AI分析提示词
```


## WebRooter 源码架构深度分析

### 触发背景
用户要求深入分析 WebRooter 的源码架构，并与 OpenClaw 现有的搜索/抓取流程进行对比。

### 项目概览
- **版本**: v0.2.2
- **语言**: Python 3.10+
- **定位**: 面向 AI 代理调用的可引用搜索 CLI
- **许可证**: MIT
- **核心输出**: `citations` + `references_text` (强制引用追溯)

### 架构总览
```
┌─────────────────────────────────────────────────────────────┐
│                        WebRooter (wr)                        │
├─────────────────────────────────────────────────────────────┤
│  CLI Layer        │  main.py (126KB) - 命令解析与分发        │
├───────────────────┼─────────────────────────────────────────┤
│  Core Engine      │  core/ - 50+ 模块                        │
│                   │  • browser.py (66KB) - Playwright封装     │
│                   │  • crawler.py (36KB) - 爬虫引擎           │
│                   │  • research_kernel.py (44KB) - 研究核心   │
│                   │  • workflow.py (47KB) - 工作流编排        │
├───────────────────┼─────────────────────────────────────────┤
│  Agents           │  agents/                                  │
│                   │  • spider.py (29KB) - 爬虫Agent           │
│                   │  • web_agent.py (100KB) - 网页Agent       │
├───────────────────┼─────────────────────────────────────────┤
│  Infrastructure   │  • cache.py (19KB) - 双层缓存              │
│                   │  • job_system.py (18KB) - 后台作业        │
│                   │  • scheduler.py (33KB) - 调度器           │
│                   │  • metrics.py (18KB) - 遥测监控           │
├───────────────────┼─────────────────────────────────────────┤
│  Output           │  • citation.py - 引用生成                  │
│                   │  JSON: {citations[], references_text}     │
└───────────────────┴─────────────────────────────────────────┘
```

### 核心设计亮点

#### 1. 强制引用机制 (Citation System)
```python
输出格式:
{
  "citations": [
    {"id": "W1", "title": "...", "url": "..."}
  ],
  "references_text": "[W1] Title https://...",
  "comparison": {
    "total_results": 8,
    "corroborated_results": 3  # 多源验证
  }
}
```
**关键设计**: AI 必须输出 `references_text`，不能直接给无来源结论。

#### 2. 双层缓存架构
```python
CACHE_MEMORY_MAX_ENTRIES = 128
CACHE_MEMORY_MAX_BYTES = 32MB      # 内存层
CACHE_SQLITE_MAX_ENTRIES = 5000
CACHE_SQLITE_BODY_MAX_BYTES = 2MB  # SQLite层
```

#### 3. 反检测/隐身系统 (StealthConfig)
- 随机 User-Agent
- Canvas 指纹噪声
- WebRTC 禁用（防IP泄露）
- 屏幕分辨率随机化
- Cloudflare 自动处理

#### 4. 命令 IR (中间表示)
```python
# core/command_ir.py
命令 -> 中间表示 -> 执行计划
# 支持干跑模式: --dry-run
```

#### 5. 后台作业系统
```bash
wr do-submit "长任务" --strict
wr jobs --status=running
wr job-status <job_id>
wr job-result <job_id>
```

### 与 OpenClaw 现有流程对比

| 维度 | WebRooter | OpenClaw (kimi-search) |
|------|-----------|------------------------|
| 定位 | 专用联网执行层 | 通用搜索工具 |
| 输出格式 | 强制 JSON + citations | Markdown/原始结果 |
| 引用追溯 | ✅ 内置 `references_text` | ❌ 需手动处理 |
| 浏览器支持 | ✅ Playwright 兜底 | ❌ 纯 HTTP |
| 反爬能力 | ✅ Stealth 配置 | ❌ 无 |
| 后台任务 | ✅ 内置 job_system | ✅ cron/subagent |
| 缓存机制 | ✅ 双层缓存 | ❌ 无 |
| 多源验证 | ✅ `corroborated_results` | ❌ 单源 |
| 工作流编排 | ✅ `skills -> do-plan -> do` | ❌ 无 |

### OpenClaw 可借鉴的设计

1. **引用追溯机制**: 为 `kimi_search` 添加 `citations` 包装层
2. **干跑模式 (Dry Run)**: `openclaw plan "任务"` 预览工具调用序列
3. **双层缓存**: 内存+SQLite缓存避免重复API调用
4. **隐身/反爬**: 复杂页面抓取时启用 stealth 配置

### 整合建议

| 场景 | 推荐方案 |
|------|----------|
| 快速搜索 + 需强制引用 | **WebRooter** (citations内置) |
| 通用工具调用 + 简单搜索 | **OpenClaw kimi-search** |
| 复杂网页抓取 + 反爬 | **WebRooter** (stealth) |
| 多 Agent 协作 + 工作流 | **OpenClaw** (更完整的 Agent 架构) |
| 生产环境审计追溯 | **WebRooter** (references_text) |

## Deep Research Skill 开发 (03-16 01:45)

### 项目决策
将 WebRooter 的核心研究能力封装成 OpenClaw Skill，而非整合到主 Agent。

**决策理由**:
1. 符合 OpenClaw 分层架构 (Skill Layer 懒加载)
2. WebRooter 是完整垂直功能，适合 Skill 边界
3. 独立维护、版本管理
4. 多 Agent 可复用

### Skill 文件结构
```
skills/deep-research/
├── SKILL.md              # 技能文档
├── __init__.py           # 入口: run(), research(), run_stream()
├── research_engine.py    # 核心引擎 (13KB)
├── cache.py              # 双层缓存 (7KB)
├── citation.py           # 引用生成 (3KB)
├── config.py             # 配置管理
└── examples.py           # 使用示例
```

### 核心功能

| 模式 | 功能 | 输出 |
|------|------|------|
| `quick` | 仅搜索摘要 | 快速摘要 + 引用 |
| `standard` | 搜索 + 抓取 | 详细答案 + 引用 |
| `deep` | MindSearch 风格 | 综合报告 + 探索路径 + 引用 |

### 技术特性

**双层缓存**:
- 内存层: LRU, 128条目, 32MB
- SQLite层: 持久化, 5000条目

**智能抓取**:
- HTTP 优先 (kimi_fetch)
- 失败自动回退 Browser (browser_snapshot)

**引用系统**:
- citations: `[{"id": "S1", "title": "...", "url": "..."}]`
- references_text: `[S1] Title (domain) url`

### 使用方式

```python
from skills.deep_research import run, research

# 方式1: Skill 接口
result = await run({
    "query": "AI agent frameworks 2026",
    "depth": "deep"
})

# 方式2: 直接调用
result = await research("Python best practices", depth="quick")

# 输出
result.answer           # 综合答案
result.references_text  # 可引用参考文献
result.citations        # 结构化引用
```

### 环境变量
```bash
DEEP_RESEARCH_CACHE_TTL=3600         # 缓存过期时间
DEEP_RESEARCH_CACHE_SIZE=128         # 内存缓存大小
DEEP_RESEARCH_CACHE_PATH=~/.openclaw/deep_research_cache.db
DEEP_RESEARCH_MAX_DEPTH=3            # 最大研究深度
DEEP_RESEARCH_AUTO_FALLBACK=true     # HTTP失败时自动使用Browser
```
