## 晚间讨论：方案修正与技术原理澄清

### 用户反馈：方案过于理想化

**被指出的问题**:
1. 技术可行性过于乐观（蓝牙+雷达融合精度、TinyML准确率）
2. 逻辑矛盾（隐私优先 vs 云端大模型、开源 vs 用小米设备验证）
3. 遗漏关键工程问题（冷启动、安装复杂度、故障处理、家庭成员接受度）
4. 与成熟方案竞争力不明（小米有3000人AI团队+10亿设备数据）

**修正后的务实定位**:
- 承认局限的**研究型项目**：探索多传感器融合的可行性
- 或**极简MVP**：只做"人走关灯"一个核心场景
- 或**集成现有生态**：HA+OpenClaw组合，OpenClaw只负责友好通知

### 关键技术发现：OpenClaw Skills原理

**重要发现**: Skills不是代码库，而是**声明式指令文件**

```
Skill文件结构:
├── YAML Frontmatter (元数据：名称、描述、环境变量要求)
└── 自然语言指令 (告诉AI如何调用工具)

执行流程:
用户请求 → AI读取Skill → AI生成工具调用 → Gateway执行 → 返回结果
```

**Home Assistant集成真相**:
- 已有社区 `homeassistant` skill
- 配置 `HA_URL` + `HA_TOKEN` 即可使用
- AI通过**自然语言指令**学会调用HA REST API
- 不是程序调用，而是**AI推理后生成curl命令**

**对项目的影响**:
- ✅ 不需要自己写HA集成代码
- ✅ 复用HA成熟的设备控制能力
- ✅ 我们的价值在"决策逻辑"而非"执行机制"

### 修正后的项目架构

```
感知层: 蓝牙Beacon + 毫米波雷达 + PIR (我们的创新)
    ↓
边缘层: TinyML推理 (我们的创新，但准确率需实测)

认知层: 大模型场景理解 (我们的创新)
    ↓
执行层: Home Assistant API (复用成熟方案，不重复造轮子)
```

**核心创新点重新聚焦**:

**不声称的**:

## Vibero Reader 项目开发

### 项目背景
用户需要双语小说阅读器，支持：
- 整本书完整翻译
- 人物/概念卡片自动生成
- 交互式双语界面

### 技术方案演进

#### Phase 1: Flask后端方案
创建完整的三层架构：
```
backend/
├── app.py              # Flask API（分析、翻译、术语提取）
├── templates/
│   └── index.html      # 三栏式Vibero风格界面
├── start.sh/start.bat  # 跨平台启动脚本
└── DEPLOY.md           # 部署文档
```

**关键修复**:
- 使用 `send_file()` 替代 `render_template()` 解决模板路径问题
- 实现 `escapeRegExp()` 防止中文正则匹配错误
- 添加卡片式悬停提示系统

#### Phase 2: 整书翻译功能
新增功能：
- `/api/translate-book` - 批量翻译所有章节
- `/api/extract-book-terms` - 从全书提取术语（每章前3段，最多15段样本）
- 🔥 Translate All 按钮 + 进度显示

#### Phase 3: 独立处理器（无服务器方案）
用户希望无需配置服务器，直接生成独立HTML。

**novel_processor.py** - 云端API版：
- 使用 Kimi/OpenAI API
- 完整全书分析、术语提取、翻译
- 生成自包含的双语阅读器HTML

**novel_processor_local.py** - 本地模型版：
- 使用 Ollama + llama3.1
- 完全免费，隐私安全
- 无需API Key

### 使用流程

**云端版**:
```bash
python novel_processor.py
# 输入: API Key + 小说TXT
# 输出: 小说名_bilingual.html
```

**本地版**:
```bash
# 1. 安装 Ollama
ollama pull llama3.1

# 2. 运行
python novel_processor_local.py
# 输出: 小说名_local.html
```

### 输出界面功能
- 三栏布局：章节列表 | 双语阅读 | 人物/概念详情
- 视图切换：双语/中文/英文
- 术语高亮：hover显示预览，点击查看详情
- 卡片系统：人物头像+身份、概念分类+描述
