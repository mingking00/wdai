## 集成概览

MemRL (Memory Reinforcement Learning) 已成功集成到 OpenClaw 系统中。系统现在具备：

4. **增强Coordinator** - 任务自动追踪和反馈

## 工作流程

### 1. 任务分配流程

```
用户请求: "部署博客到GitHub"
        │
        ▼
┌─────────────────────────────────────┐
│ Coordinator.assign_task()           │
│  1. 调用 memory_search_q("部署")    │
│  2. 检索相关记忆 (带Q值排序)        │
│  3. 分配给Coder Agent              │
│  4. 开始任务追踪                    │
└─────────────────────────────────────┘
        │
        ▼
Coder Agent 执行任务
        │
        ▼
完成任务 → report_task_complete()
        │
        ▼
┌─────────────────────────────────────┐
│ 自动更新Q值                         │
│  1. 计算奖励 (成功=1.0)            │
│  2. 更新相关记忆的Q值              │
│  3. 如成功，添加新经验             │
│  4. 触发Reflector Agent            │
└─────────────────────────────────────┘
```

### 2. 记忆检索流程 (两阶段)

```
查询: "部署博客"
        │
        ▼
阶段A: 语义召回
  • 计算查询与所有记忆的余弦相似度
  • 筛选相似度 > 0.5 的候选 (Top 20)
        │
        ▼
阶段B: Q值重排
  • 对每个候选计算: 得分 = 0.5×相似度 + 0.5×Q值
  • 按得分排序 (Top 5)
        │
        ▼
返回: 按综合得分排序的记忆列表
```

### 3. Q值更新流程

```
任务完成 → 奖励计算
  • 1.0 = 成功 + 已验证
  • 0.9 = 成功 + 用户确认
  • 0.7 = 成功 (未验证)
  • 0.0 = 失败
        │
        ▼
Q值更新公式:
  Q_new = Q_old + α × (reward - Q_old)
  (α = 0.1, 学习率)
        │
        ▼
结果:
  • 成功经验 → Q值上升
  • 失败经验 → Q值下降
  • 多次使用后收敛到真实成功率
```

```python
# 检索记忆 (自动使用Q值排序)
```

## 配置参数

文件: `.claw-status/memrl_config.json`

```json
{
  "alpha": 0.1,              // Q值学习率 (0.05-0.2)
  "lambda_weight": 0.5,      // 语义vs效用权重 (0-1)
  "similarity_threshold": 0.5,  // 语义召回阈值
  "top_k_candidates": 20,    // 阶段A召回数量
  "top_k_final": 5,          // 阶段B最终数量
  "initial_q": 0.5,          // 新记忆初始Q值
  "min_q": 0.1,              // Q值下限
  "max_q": 1.0               // Q值上限
}
```

### 参数调优建议

| 参数 | 调优方向 | 效果 |
|------|---------|------|
| `alpha` | 增大(0.2) | Q值更新更快，适合快速变化环境 |
| `alpha` | 减小(0.05) | Q值更稳定，适合稳定环境 |
| `lambda_weight` | 增大(0.7) | 更信任历史成功率，适合成熟领域 |
| `lambda_weight` | 减小(0.3) | 更信任语义匹配，适合新领域 |
