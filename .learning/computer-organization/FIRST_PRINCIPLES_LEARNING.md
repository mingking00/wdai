## 🏗️ 第一性原理拆解

### 问题: 计算机是什么?

**从物理层面**:
```
晶体管 (电子开关)
    ↓
逻辑门 (Nand/And/Or) 
    ↓
算术单元 (加法器/ALU)
    ↓
存储单元 (寄存器/RAM)
    ↓
控制单元 (CPU)
    ↓
计算机 (执行指令的机器)
```

**核心原理**:
1. **信息表示**: 所有信息用二进制表示 (0和1)
2. **逻辑运算**: 用逻辑门操作二进制
3. **存储**: 用触发器存储状态
4. **控制**: 用有限状态机控制执行流程
5. **指令**: 程序是指令序列，指令操作数据

**这就是不变的地基。**

无论是:
- 50年前的Intel 4004
- 现在的Apple M3芯片  
- 还是未来的量子计算机(可能)

**底层原理是一样的**: 存储、运算、控制。

## 📚 原理导向的学习路径

### Stage 0: 计算机的本质 (Nand2Tetris Part I)

**不是**: "学会用HDL写逻辑门"  
**而是**: "理解从沙子到计算机的过程"

| 主题 | 核心原理 | 技术细节(次要) |
|------|----------|----------------|
| 布尔逻辑 | 信息可以用二进制表示和运算 | HDL语法 |
| 算术单元 | 加法可以由逻辑门实现 | 具体电路图 |
| 存储 | 状态可以用触发器保持 | RAM芯片型号 |
| CPU | 指令驱动数据通路 | x86/ARM指令集 |
| 汇编 | 人类可读的机器语言 | 具体汇编语法 |

### Stage 1: 程序的本质 (CSAPP)

**核心问题**:
- 程序是怎么被编译的?
- 程序在内存中是什么样?
- 为什么有栈和堆?
- 函数调用怎么工作的?
- 缓存为什么重要?

**理解这些 = 理解任何语言的基础**

### Stage 2: 操作系统的本质

**核心问题**:
- 什么是进程? (执行的程序)
- 什么是虚拟内存? (地址空间的抽象)
- 什么是文件系统? (持久化存储的组织)
- 什么是并发? (多任务切换)

### Stage 3: 回到Linux内核

**此时再看内核**:
- 不再是"提取技巧"
- 而是"验证原理"
- 理解"为什么这样设计"

## 🎯 实践原则

### 学习每个概念时:

**1. 问"为什么"5次**
```
例: 为什么用补码?
→ 为了统一加减法
→ 为什么能统一?
→ 因为补码的数学性质
→ 什么性质?
→ -x = ~x + 1 (按位取反加1)
→ 为什么这个性质成立?
→ (数学推导...)
```

**2. 用"自己的话"解释**
- ❌ 不要: 背诵定义
- ✅ 要: 向别人解释清楚
- 如果解释不清 = 没真懂

**3. 关注"不变的东西"**
- 布尔代数 (50年前和50年后一样)
- 冯·诺依曼架构 (70年没变)
- 栈的工作方式 (所有语言通用)

**4. 技术是例子，不是目标**
- HDL是实现原理的工具
- x86是理解架构的例子
- Linux是OS原理的具体化

## 💡 关于编程语言的地基

**理解计算机原理 = 理解所有语言的根基**

| 语言特性 | 底层原理 | 例子 |
|----------|----------|------|
| 变量 | 内存存储 | 栈/堆分配 |
| 函数调用 | 栈帧、跳转 | call/ret指令 |
| 条件语句 | 条件跳转 | if→cmp+jmp |
| 循环 | 跳转 | while→jmp |
| 对象 | 内存布局 | vtable、指针 |
| 并发 | 上下文切换 | 寄存器保存/恢复 |
| GC | 内存管理 | 标记-清除、引用计数 |

**无论Python/C/Java/Rust，底层都是这些原理。**
