# Skill Evolution Framework - 技能进化框架
# 内化时间: 2026-03-10

## 核心理念 (来自用户洞察)

用户指出：技能进化到顶级需要三个核心要素：
1. **归纳小技巧** —— 从实践中提取可复用的模式
2. **创造思想工具** —— 将技巧组合成认知框架
3. **实践验证** —— 不断验证工具的高效性和准确性

我补充了两个维度：
4. **教授他人** —— 通过教学检验理解深度
5. **跨域迁移** —— 提取通用原则，应用到其他领域

---

## 进化阶段

```
新手(Novice)        → 学习规则
熟练(Competent)     → 应用规则  
精通(Proficient)    → 识别模式 ← 归纳小技巧
专家(Expert)        → 创造工具 ← 组合技巧
大师(Master)        → 直觉决策 ← 内化工具
```

## 工作流程

### 阶段1: 归纳小技巧
**何时**: 每个项目结束后
**做什么**:
- 回顾项目中的关键解决方案
- 提取3-5个可复用技巧
- 记录: 场景 + 模式 + 代码 + 效率提升
- 追踪: 验证次数和成功率

**数据结构** (SkillTip):
```python
{
    id: "tip-20260310-123456",
    name: "Async+Batch Pattern",
    context: "优化Agent响应时间",
    pattern: "批处理+异步并发",
    code_example: "...",
    efficiency_gain: "吞吐量提升3x",
    verified_count: 5,
    success_rate: 0.95
}
```

**工具**:
```python
engine.extract_tip(task, solution, result)
engine.verify_tip(tip_id, new_task, success)
```

---

### 阶段2: 创造思想工具
**何时**: 有3+个相关技巧且验证次数≥3
**做什么**:
- 将相关技巧组合成工具
- 定义: 适用场景 + 使用步骤
- 命名: 描述性名称 (如 "Resilient Multi-Agent Orchestrator")

**数据结构** (MentalTool):
```python
{
    id: "tool-20260310-123456",
    name: "Resilient Orchestrator",
    description: "组合投票+批处理+熔断器",
    applicability: ["多智能体系统", "分布式协调"],
    steps: ["应用投票机制", "批处理优化", "添加熔断保护"],
    tips_used: ["tip-1", "tip-2", "tip-3"],
    verification_log: [...]
}
```

**工具**:
```python
tool = engine.create_mental_tool("ToolName", [tip1.id, tip2.id])
engine.use_tool(tool.id, context)
engine.verify_tool(tool.id, success, feedback)
```

---

### 阶段3: 实践验证
**何时**: 持续进行
**做什么**:
- 每个技巧必须验证3次以上
- 追踪真实成功率
- 记录成功/失败场景
- 成功率<70%的技巧标记为需改进

**验证原则**:
- 不是"我觉得有效"
- 而是"数据证明有效"
- 失败案例比成功案例更有学习价值

---

### 阶段4: 教授他人 (费曼技巧)
**何时**: 每季度
**做什么**:
- 生成技能教学指南
- 检验能否清晰解释
- 发现理解中的gap
- 完善知识盲区

**交付物**:
- teaching_guide.md
- 包含: 工具介绍 + 适用场景 + 使用步骤 + 底层技巧

---

### 阶段5: 跨域迁移
**何时**: 工具成熟后 (验证>10次，成功率>80%)
**做什么**:
- 提取领域无关的核心原则
- 识别可应用的其他领域
- 在新领域应用并记录效果

**示例**:
```
原始: 多智能体系统中的投票机制
抽象: 分布式决策中的共识达成
迁移: 团队决策、数据融合、推荐系统
```

## 核心洞察

**不是简单的记录**:
❌ "我做过X项目"

**而是系统化的进化**:
✅ "从X项目提取技巧A，验证5次后组合成工具B，应用到Y场景"

**关键差异**:
- 被动记录 → 主动提取
- 经验积累 → 模式识别
- 单点知识 → 工具网络
- 个人技能 → 可迁移原则
