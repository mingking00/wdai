## 3. 关键技术机制

### 3.1 记忆提取 (Extraction)
```python
# 两阶段提取
Phase 1: 原始消息 → 候选记忆候选 (LLM提取)
Phase 2: 冲突解决 → 更新/合并/删除 (ReAct循环)
```

**冲突解决策略**：
- 新事实覆盖旧事实
- 矛盾事实标记待确认
- 相似事实合并

### 3.2 记忆存储结构
```json
{
  "memory_id": "uuid",
  "user_id": "user_123",
  "memory_text": "用户偏好Python",
  "embedding": [0.1, 0.2, ...],
  "metadata": {
    "category": "preference",
    "confidence": 0.95,
    "created_at": "2026-03-10",
    "last_accessed": "2026-03-10",
    "decay_score": 0.8  // 衰减分数
  }
}
```

### 3.3 检索策略 (4种模式)

| 策略 | 机制 | 适用场景 |
|------|------|----------|
| **Dump All** | 加载全部记忆 | <20条记忆 |
| **Semantic Search** | 语义最近邻 | 大记忆库 |
| **Pre-loaded** | 会话开始时加载 | 跨会话记忆 |
| **Hybrid** | 预加载+条件搜索 | 生产环境 |

## 4. 与简单 RAG 的区别

| 维度 | 简单 RAG | Mem0 |
|------|----------|------|
| 数据类型 | 静态文档 | 动态交互 |
| 更新频率 | 批量索引 | 实时提取 |
| 冲突处理 | 无 | 智能合并 |
| 衰减机制 | 无 | 时间衰减 |
| 个性化 | 无 | 用户级隔离 |
| 置信度 | 无 | 置信评分 |

**关键洞察**：
> RAG = 检索外部知识
> Mem0 = 管理内部状态

## 5. 三层记忆模型 (内化到认知架构)

```
┌─────────────────────────────────────────┐
│         Working Memory (工作记忆)         │
│    - 当前会话上下文                       │
│    - 近期交互 (Context Window)           │
└─────────────────────────────────────────┘
                   ↕
┌─────────────────────────────────────────┐
│         Episodic Memory (情景记忆)        │
│    - 会话历史摘要                         │

│    - 用户交互轨迹                         │
└─────────────────────────────────────────┘
                   ↕
┌─────────────────────────────────────────┐
│         Semantic Memory (语义记忆)        │
│    - 用户偏好                            │
│    - 领域知识                            │
│    - 长期事实                            │
└─────────────────────────────────────────┘
```

**与我的三层记忆对应**：
- Working = 当前对话上下文
- Episodic = MEMORY.md (经验记录)
- Semantic = SOUL.md (内化原则)

## 6. 核心设计原则 (可复用)

### 原则1: 原子化存储
- 记忆拆分为最小事实单元
- 便于部分更新和检索

### 原则2: 置信度评分
- 每条记忆有置信分数
- 低置信度标记待验证

### 原则3: 时间衰减
- 旧记忆重要性递减
- 新记忆优先召回

### 原则4: 用户隔离
- 严格的 user_id 隔离
- 防止记忆泄漏

### 原则5: 元数据丰富
- category, tags, timestamp
- 支持多维度过滤

## 7. 失败教训 (从研究中发现)

**自建记忆的陷阱**：
1. 简单语义搜索丢失细微语境
2. 新旧偏好冲突无法解决
3. 重复数据无限膨胀
4. 缺乏衰减机制导致旧记忆干扰

**Mem0 的解决方案**：
- 智能提取 (不只是存储)
- 冲突解决 (不只是追加)
- 自动去重 (不只是索引)
- 动态衰减 (不只是保留)

## 11. 关键代码模式

### 记忆提取 Prompt 模板
```python
EXTRACTION_PROMPT = """
从以下对话中提取原子化记忆事实。
每条记忆应该是一个独立、完整的事实。

规则：
1. 提取用户偏好、关键决策、约束条件
2. 忽略临时性、过程性内容
3. 标记置信度 (high/medium/low)
4. 检测与现有记忆的矛盾

对话：
{conversation}

现有记忆：
{existing_memories}

输出格式：
[
  {
    "fact": "用户偏好...",
    "category": "preference|constraint|decision|todo",
    "confidence": "high|medium|low",
    "conflicts_with": null | [memory_id]
  }
]
"""
```

### 检索算法
```python
def retrieve_memories(query, user_id, k=5):
    # 1. 向量化查询
    query_emb = embed(query)
    
    # 2. 语义搜索
    candidates = vector_search(query_emb, user_id, k=k*2)
    
    # 3. 应用衰减权重
    for mem in candidates:
        mem['score'] *= decay_factor(mem['last_accessed'])
    
    # 4. 重新排序
    candidates.sort(key=lambda x: x['score'], reverse=True)
    
    # 5. 返回 top-k
    return candidates[:k]
```
