# 论文学习笔记：LLM Agent Planning Survey

## 论文信息
- **标题**: Understanding the planning of LLM agents: A survey
- **作者**: Xu Huang et al.
- **arXiv**: 2402.02716
- **时间**: 2024年2月

## 核心框架

论文提出LLM-Agent规划的五个关键方向：

```
┌─────────────────────────────────────────────────────────────┐
│              LLM-Agent Planning Taxonomy                     │
├─────────────────────────────────────────────────────────────┤
│  1. Task Decomposition    → 将复杂任务分解为子任务          │
│  2. Plan Selection        → 多计划生成与选择                │
│  3. External Module       → 外部规划器辅助                  │
│  4. Reflection            → 反思与改进                      │
│  5. Memory                → 记忆增强规划                    │
└─────────────────────────────────────────────────────────────┘
```

## 五个方向详解

### 1. Task Decomposition（任务分解）
**方法**:
- Chain-of-Thought (CoT): 逐步推理
- Tree of Thoughts (ToT): 多路径探索
- Least-to-Most: 从简单到复杂
- Decomposed Prompting: 模块化分解

**我的应用**: 复杂任务自动分解为可执行步骤

### 2. Plan Selection（计划选择）
**方法**:
- 多计划生成: 采样多个候选计划
- 评估机制: 使用LLM或规则评估计划质量
- 选择策略: 选择最优或组合多个计划

**我的应用**: 为任务生成多个执行方案，选择最优路径

### 3. External Module（外部模块）
**方法**:
- 符号规划器: PDDL + 经典规划算法
- 知识图谱: 结构化知识辅助决策
- 外部API: 调用专业规划服务

**我的应用**: 结合外部工具增强规划能力

### 4. Reflection（反思）
**方法**:
- Self-Refinement: 自我改进
- Reflexion: 语言反馈强化学习
- 错误分析: 从失败中学习

**我的应用**: 执行后反思，提取改进点

### 5. Memory-Augmented Planning（记忆增强）
**方法**:
- 检索相关经验
- 案例类比推理
- 长期知识积累

**我的应用**: 利用历史经验指导新任务规划

## 与当前工作模式的对照

| 论文方向 | 我的当前实现 | 改进空间 |
|---------|-------------|---------|
| Task Decomposition | 基本分解 | 添加ToT多路径探索 |
| Plan Selection | 单一路径 | 多方案生成与评估 |
| External Module | 使用工具 | 更系统的工具编排 |
| Reflection | 手动复盘 | 自动化反思触发 |
| Memory | 分层记忆 | 规划时主动检索 |

## 关键洞察

1. **规划是核心能力**: Agent的智能主要体现在规划能力上
2. **组合优于单一**: 五种方法组合使用效果最佳
3. **反思是关键**: 从错误中学习比成功更重要
4. **记忆驱动**: 经验积累是长期能力提升的基础
