# 🛡️ 灵感进化引擎 - 风险评估标准与决策清单

## 风险评估框架

系统采用**五维度风险评估模型**，综合评分决定是否可以自动实施。

### 评估维度

| 维度 | 权重 | 说明 | 评分标准 |
|------|------|------|----------|
| **影响范围** | 25% | 修改会影响多少文件/功能 | 单文件10分 → 系统范围60分 → 基础设施80分 |
| **关键程度** | 30% | 修改的是核心还是边缘功能 | 实验性10分 → 核心逻辑70分 → 调度系统85分 |
| **实施复杂度** | 20% | 修改的复杂程度 | 配置修改10分 → 重构40分 → 架构变更70分 |
| **可逆性** | 15% | 出问题时能否快速回滚 | 完全可逆10分 → 部分可逆50分 → 破坏性变更80分 |
| **测试覆盖** | 10% | 修改是否容易验证 | 自动可验证10分 → 难测试70分 |

### 风险等级划分

```
评分 ≤ 35分: 🟢 低风险 (LOW)
   → 自动实施，无需审批

35分 < 评分 ≤ 65分: 🟡 中等风险 (MEDIUM)
   → 生成决策清单，等待你的确认

评分 > 65分: 🔴 高风险 (HIGH)
   → 必须人工评审，生成详细评审文档
```

## 配置选项

### 修改审批策略

编辑 `evolution_engine.py` 中的初始化参数:

```python
evolution = EvolutionEngine(
    require_approval_for_medium=True  # 中风险是否需要审批
)
```

- `True` (默认): 中风险方案需要你的确认
- `False`: 中风险方案也会自动实施（不推荐）

### 风险阈值调整

编辑 `risk_assessment.py` 中的阈值:

```python
THRESHOLDS = {
    'low_max': 35,      # 低风险上限
    'medium_max': 65,   # 中等风险上限
}
```
