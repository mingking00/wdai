## s09: L2/L3 安全分析

### 目标
从简单的正则匹配升级到 AI 辅助的复杂漏洞检测。

### 架构
```
Security Agent
  ├── L1: Fast Check (0.1ms) - 53 条规则 ✅ 已有
  ├── L2: Semgrep (1-10s) - 复杂模式 📋 新增
  └── L3: AI Review (5-30s) - 业务逻辑 📋 新增
```

### 关键技术
- **L2**: Semgrep Registry + 自定义规则
- **L3**: LLM 上下文感知分析
- **协同**: 智能调度，根据风险分数选择深度

### 触发策略
| 风险分数 | 执行层 | 时间 |
|:---:|:---|:---:|
| < 0.3 | L1 | 0.1ms |
| 0.3-0.7 | L1 + L2 | 1-10s |
| > 0.7 | L1 + L2 + L3 | 5-30s |

## s10: 自动会话摘要

### 目标
每次对话结束后自动提取关键信息并生成结构化摘要。

### 组件
1. **SessionTracker**: 实时追踪关键事件
2. **KeyInfoExtractor**: AI 辅助提取关键信息
3. **Summarizer**: 生成 brief + detailed + full 摘要
4. **PeriodicSummary**: 自动生成周报/月报

### 输出示例
```markdown
# Session Summary - 2026-03-18

## Tasks
1. ✅ 批量导入 Semgrep 规则

## Key Decisions
| Decision | Rationale |
|:---|:---|
| 使用本地生成 | 网络慢，本地更可靠 |

## Errors
| Error | Fix |
|:---|:---|
| API rate limit | 切换到本地生成 |

## Learnings
- 借鉴不是照搬
```

## s11: 自适应学习率

### 目标
根据任务成功率自动调整学习参数，优化探索与利用。

### 核心算法
- **成功率追踪**: 指数加权移动平均
- **参数调整**: 根据成功率动态设置探索率、学习率、重试次数
- **策略选择**: UCB1 Bandit 算法
- **学习率**: 根据收敛情况自动调整

### 参数矩阵
| 场景 | exploration_rate | learning_rate | retry | timeout |
|:---|:---:|:---:|:---:|:---:|
| 新任务 | 0.5 | 0.05 | 5 | 120s |
| 成功任务 | 0.1 | 0.2 | 2 | 30s |
| 困难任务 | 0.4 | 0.15 | 4 | 90s |
| 平衡 | 0.3 | 0.1 | 3 | 60s |
