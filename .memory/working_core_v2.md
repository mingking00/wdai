## 检查点清单（自动暂停等待调整）

### 检查点0: 问题理解
- **触发**: 收到新任务
- **系统动作**: 分析问题类型，提议执行路径
- **暂停**: 等待用户确认或调整方向
- **用户选项**:
  - "继续" → 进入分配器
  - "调整方向" → 重新定义问题
  - "直接S2" → 跳过S1广域搜索
  - "简化处理" → 降级为常规回复

### 检查点1: S1广域搜索前
- **触发**: 确定需要搜索
- **系统动作**: 展示搜索策略（关键词/来源/筛选标准）
- **暂停**: 等待确认搜索范围
- **用户选项**:
  - "继续" → 执行搜索
  - "缩小范围" → 指定特定领域
  - "扩大范围" → 增加语言/时间跨度
  - "跳过S1" → 直接进入S2

### 检查点2: S1结果筛选后
- **触发**: 完成广域搜索
- **系统动作**: 展示筛选结果 + 可信度评估 + 可复现性检查
- **暂停**: 等待确认是否足够
- **用户选项**:
  - "继续S2" → 进入进化循环
  - "补充搜索" → 针对盲区再搜索
  - "直接执行" → 用现有结果解决问题
  - "放弃此方向" → 重新搜索其他方向

### 检查点3: S2进化循环每轮后
- **触发**: 每完成一轮内化/执行/分析
- **系统动作**: 展示当前进展 + 解决率 + 剩余盲区
- **暂停**: 等待确认是否继续循环
- **用户选项**:
  - "继续循环" → 下一轮进化
  - "调整策略" → 改变搜索重点（论文→代码→Skill）
  - "深度分析" → 当前结果第一性原理分析
  - "暂停记录" → 保存当前进度，后续继续
  - "停止" → 输出当前最佳方案，退出循环

### 检查点4: S3固化前
- **触发**: 任务完成且可复现
- **系统动作**: 展示拟创建的Skill设计 + 写入位置
- **暂停**: 等待确认固化方式
- **用户选项**:
  - "固化" → 创建Skill并写入核心
  - "优化设计" → 调整Skill接口/参数
  - "暂缓" → 记录为待固化任务
  - "不固化" → 仅记录执行日志

## 自动检测触发词

当用户说以下词语时，**立即暂停当前循环**，进入调整模式：
- "调整" / "修改" / "不对" / "错了"
- "等等" / "停下" / "暂停"
- "换方向" / "重新"
- "简单说" / "太复杂"

```
       ↓
系统: S2进化循环 → 内化 → 执行测试 → [检查点3] 第一轮: A方案80%解决
       
用户: "继续循环" / "调整策略找B的代码" / "停止用A方案"
       ↓
系统: 根据选择继续/调整/停止...
```
